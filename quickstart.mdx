---
title: "Get started with Buildkite Agent"
description: "Start your first Buildkite Agent in 4 steps — install the agent, grab your token, configure build paths, and connect to Buildkite."
---

This guide walks you through installing the Buildkite Agent, retrieving your agent token, and starting your first agent so it appears in the Buildkite dashboard ready to accept jobs.

<Steps>
  <Step title="Install the agent">
    Choose the installation method for your platform and follow the instructions on the [installation page](/installation).

    <CodeGroup>

    ```bash macOS (Homebrew)
    brew install buildkite/buildkite/buildkite-agent
    ```

    ```bash Linux (shell installer)
    bash -c "`curl -sL https://raw.githubusercontent.com/buildkite/agent/main/install.sh`"
    ```

    ```powershell Windows (PowerShell, elevated)
    $env:buildkiteAgentToken = "your-agent-token"
    iex ((New-Object System.Net.WebClient).DownloadString(
      'https://raw.githubusercontent.com/buildkite/agent/main/install.ps1'
    ))
    ```

    ```bash Docker
    docker pull buildkite/agent:3-ubuntu-22.04
    ```

    </CodeGroup>

    Confirm the binary is available before continuing:

    ```bash
    buildkite-agent --version
    ```
  </Step>

  <Step title="Get your agent token">
    Your agent token is how the agent authenticates with Buildkite. To find it:

    1. Log in to [buildkite.com](https://buildkite.com) and open your organization.
    2. Click **Agents** in the top navigation.
    3. Copy the token shown in the **Agent token** section.

    <Warning>
      Treat your agent token like a password. Anyone with this token can register agents that run jobs in your organization. Do not commit it to source control.
    </Warning>
  </Step>

  <Step title="Start the agent">
    Pass your token and a local directory for build checkouts to `buildkite-agent start`:

    ```bash
    buildkite-agent start --token=<your-token> --build-path=/tmp/buildkite-builds
    ```

    <Note>
      `--build-path` must be a directory writable by the user running the agent. Create it first if it does not exist: `mkdir -p /tmp/buildkite-builds`.
    </Note>

    When started, the agent:

    1. **Registers** a new agent identity with Buildkite using the token
    2. **Pings** the Buildkite API on a regular interval looking for scheduled jobs
    3. **Accepts** a job when one is dispatched to this agent's queue
    4. **Runs** the job bootstrap, checking out code and executing your pipeline steps
    5. **Streams** log output and artifacts back to Buildkite in real time

    You should see output similar to:

    ```text
    buildkite-agent v3.123.0 starting...
    Registering agent with Buildkite...
    Agent registered: <hostname>-1 (queue: default)
    Connecting to Buildkite...
    Waiting for work...
    ```

    <Tip>
      To run the agent in the background as a service, use your platform's service manager: `systemctl enable --now buildkite-agent` on systemd Linux, or follow the Homebrew service instructions on macOS.
    </Tip>
  </Step>

  <Step title="Verify the agent and run a test job">
    **Check the agent appears in Buildkite:**

    1. Open **Agents** in your Buildkite organization.
    2. You should see your agent listed with a green **Connected** status and its hostname as the name.

    **Trigger a test job:**

    Create a pipeline with a minimal `pipeline.yml` and trigger a build:

    ```yaml
    steps:
      - label: "Test step"
        command: "echo 'Hello from Buildkite!'"
    ```

    Once the build is triggered, the agent picks up the job, runs the command, and streams the output back. You will see the log in real time on the build page.

    <Note>
      If no agent appears within a minute, check that `--token` is correct and that the agent host has outbound HTTPS access to `agent.buildkite.com`.
    </Note>
  </Step>
</Steps>
