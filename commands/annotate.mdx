---
title: "buildkite-agent annotate — add build annotations"
description: "Create or update Markdown annotations on the Buildkite build page from within a job. Supports styles, contexts, appending, and removal via annotation remove."
---

`buildkite-agent annotate` lets you publish rich content directly onto the Buildkite build page from within a running job. Annotations are written in CommonMark-compliant Markdown with GitHub Flavored Markdown extensions. Common uses include surfacing test result summaries, links to generated artifacts, graphs, or any information that helps your team understand what happened during a build. The annotation body can be passed as a command-line argument, piped via stdin, or generated by a script. Each annotation body may be at most 1 MiB.

## annotate

```bash
buildkite-agent annotate [body] [options...]
```

<ParamField path="[body]" type="string">
  The annotation body in Markdown. Omit to read from stdin.
</ParamField>

<ParamField path="--context" type="string">
  A unique identifier for this annotation. Annotations with the same context update each other rather than creating new ones. Maximum 100 characters. Environment variable: `BUILDKITE_ANNOTATION_CONTEXT`
</ParamField>

<ParamField path="--style" type="string">
  Visual style of the annotation. One of `success`, `info`, `warning`, or `error`. Environment variable: `BUILDKITE_ANNOTATION_STYLE`
</ParamField>

<ParamField path="--append" type="bool">
  Append to the body of an existing annotation instead of replacing it. Default: `false`. Environment variable: `BUILDKITE_ANNOTATION_APPEND`
</ParamField>

<ParamField path="--priority" type="int">
  Display priority from `1` (shown last) to `10` (shown first). Default: `3`. Environment variable: `BUILDKITE_ANNOTATION_PRIORITY`
</ParamField>

<ParamField path="--scope" type="string">
  Controls where the annotation is displayed. One of `build` (shown on the build page) or `job` (shown on the specific job). Default: `build`. Environment variable: `BUILDKITE_ANNOTATION_SCOPE`
</ParamField>

<ParamField path="--job" type="string">
  The job the annotation comes from. Defaults to `BUILDKITE_JOB_ID`. Environment variable: `BUILDKITE_JOB_ID`
</ParamField>

<ParamField path="--redacted-vars" type="string">
  Pattern of environment variable names containing sensitive values. Matching secrets are redacted from the annotation body before upload. Environment variable: `BUILDKITE_REDACTED_VARS`
</ParamField>

## annotation remove

Removes a previously published annotation. The context used when creating the annotation is required to identify it.

```bash
buildkite-agent annotation remove [options...]
```

<ParamField path="--context" type="string">
  The context of the annotation to remove. Default: `default`. Environment variable: `BUILDKITE_ANNOTATION_CONTEXT`
</ParamField>

<ParamField path="--scope" type="string">
  The scope of the annotation to remove. One of `build` or `job`. Default: `build`. Environment variable: `BUILDKITE_ANNOTATION_SCOPE`
</ParamField>

<ParamField path="--job" type="string">
  The job removing the annotation. Defaults to `BUILDKITE_JOB_ID`. Environment variable: `BUILDKITE_JOB_ID`
</ParamField>

## Examples

<CodeGroup>

```bash Simple success annotation
buildkite-agent annotate "All tests passed!" --style "success"
```

```bash Warning from a file
cat annotation.md | buildkite-agent annotate --style "warning"
```

```bash Update just the style of an existing annotation
buildkite-agent annotate --style "success" --context "test-results"
```

```bash Append to an annotation
buildkite-agent annotate "- Step 1 complete" \
  --context "progress" \
  --append
```

```bash Rich Markdown with a table
buildkite-agent annotate "$(cat <<'EOF'
## Test Results

| Suite | Passed | Failed |
|---|---|---|
| Unit | 142 | 0 |
| Integration | 38 | 2 |

[Download report](artifacts/report.html)
EOF
)" --style "warning" --context "test-results"
```

```bash Dynamic annotation from a script
./scripts/generate-report | buildkite-agent annotate \
  --style "info" \
  --context "coverage-report"
```

```bash Remove an annotation
buildkite-agent annotation remove --context "progress"
```

</CodeGroup>

<Tip>
  Use distinct `--context` values when you want multiple annotations visible at
  once. If two steps annotate with the same context, the second call updates
  the first rather than creating a duplicate.
</Tip>
