---
title: "Job API env endpoints — GET, PATCH, DELETE"
description: "GET /api/current-job/v0/env returns job environment variables. PATCH sets values. DELETE removes keys. Changes are reflected immediately in the running job."
---

The env endpoints let your job scripts read and modify the environment that the agent is tracking for the current job. Changes made through `PATCH` or `DELETE` are reflected immediately — subsequent steps and hooks that run in the same job will see the updated environment. All three endpoints require a valid bearer token in the `Authorization` header.

<Note>
Certain Buildkite agent configuration variables are protected and cannot be modified or deleted through the Job API. Attempts to modify them return a `422 Unprocessable Entity` error. See [Protected variables](#protected-variables) below for the full list.
</Note>

---

## GET /api/current-job/v0/env

Returns a snapshot of all environment variables currently set for the job as a flat JSON object.

### Response

<ResponseField name="env" type="object" required>
  A map of environment variable names to their string values. Every variable currently visible to the job is included.
</ResponseField>

### Example

<CodeGroup>

```bash Request
curl --unix-socket "$BUILDKITE_AGENT_JOB_API_SOCKET" \
  --request GET \
  --header "Authorization: Bearer $BUILDKITE_AGENT_JOB_API_TOKEN" \
  http://job/api/current-job/v0/env
```

```json Response
{
  "env": {
    "BUILDKITE_BRANCH": "main",
    "BUILDKITE_BUILD_NUMBER": "42",
    "BUILDKITE_PIPELINE_SLUG": "my-pipeline",
    "MY_CUSTOM_VAR": "hello"
  }
}
```

</CodeGroup>

## PATCH /api/current-job/v0/env

Sets or updates one or more environment variables. Pass a JSON object where each key is a variable name and each value is its new string value. Variables that already exist are updated; variables that do not exist are added. Returns lists of which keys were added and which were updated.

Setting a value to `null` is not permitted on this endpoint — use `DELETE /api/current-job/v0/env` to remove variables.

### Request body

<ParamField body="env" type="object" required>
  A map of environment variable names to string values. Each key must be a non-null string value.

  <Expandable title="env properties">
    <ParamField body="env.<KEY>" type="string" required>
      The value to assign to the environment variable named `<KEY>`. Must not be `null` — use the `DELETE` endpoint to remove variables.
    </ParamField>
  </Expandable>
</ParamField>

### Response

<ResponseField name="added" type="string[]" required>
  Alphabetically sorted list of variable names that were newly created (did not exist before the request).
</ResponseField>

<ResponseField name="updated" type="string[]" required>
  Alphabetically sorted list of variable names that already existed and were overwritten with a new value.
</ResponseField>

### Example

<CodeGroup>

```bash Request
curl --unix-socket "$BUILDKITE_AGENT_JOB_API_SOCKET" \
  --request PATCH \
  --header "Authorization: Bearer $BUILDKITE_AGENT_JOB_API_TOKEN" \
  --header "Content-Type: application/json" \
  --data '{
    "env": {
      "MY_SECRET": "s3cr3t",
      "BUILDKITE_BRANCH": "feature/new-thing"
    }
  }' \
  http://job/api/current-job/v0/env
```

```json Response
{
  "added": ["MY_SECRET"],
  "updated": ["BUILDKITE_BRANCH"]
}
```

</CodeGroup>

## DELETE /api/current-job/v0/env

Removes one or more environment variables from the job. Pass a list of variable names to delete. Only variables that currently exist are removed — names that do not exist are silently ignored. Returns the sorted list of variables that were actually deleted.

### Request body

<ParamField body="keys" type="string[]" required>
  An array of environment variable names to remove. Names that do not exist in the current environment are ignored.
</ParamField>

### Response

<ResponseField name="deleted" type="string[]" required>
  Alphabetically sorted list of variable names that were removed. Variables listed in the request that did not exist in the environment are omitted from this list.
</ResponseField>

### Example

<CodeGroup>

```bash Request
curl --unix-socket "$BUILDKITE_AGENT_JOB_API_SOCKET" \
  --request DELETE \
  --header "Authorization: Bearer $BUILDKITE_AGENT_JOB_API_TOKEN" \
  --header "Content-Type: application/json" \
  --data '{
    "keys": ["MY_SECRET", "TEMP_VAR", "DOES_NOT_EXIST"]
  }' \
  http://job/api/current-job/v0/env
```

```json Response
{
  "deleted": ["MY_SECRET", "TEMP_VAR"]
}
```

</CodeGroup>

## Protected variables

The following environment variables are set by agent configuration and cannot be modified or deleted through the Job API. Sending any of these keys in a `PATCH` or `DELETE` request returns `422 Unprocessable Entity`.

| Variable |
|---|
| `BUILDKITE_AGENT_ACCESS_TOKEN` |
| `BUILDKITE_AGENT_DEBUG` |
| `BUILDKITE_AGENT_ENDPOINT` |
| `BUILDKITE_AGENT_PID` |
| `BUILDKITE_BIN_PATH` |
| `BUILDKITE_BUILD_PATH` |
| `BUILDKITE_COMMAND_EVAL` |
| `BUILDKITE_CONFIG_PATH` |
| `BUILDKITE_CONTAINER_COUNT` |
| `BUILDKITE_GIT_CLEAN_FLAGS` |
| `BUILDKITE_GIT_CLONE_FLAGS` |
| `BUILDKITE_GIT_CLONE_MIRROR_FLAGS` |
| `BUILDKITE_GIT_FETCH_FLAGS` |
| `BUILDKITE_GIT_MIRRORS_LOCK_TIMEOUT` |
| `BUILDKITE_GIT_MIRRORS_PATH` |
| `BUILDKITE_GIT_MIRRORS_SKIP_UPDATE` |
| `BUILDKITE_GIT_SUBMODULES` |
| `BUILDKITE_HOOKS_PATH` |
| `BUILDKITE_HOOKS_SHELL` |
| `BUILDKITE_KUBERNETES_EXEC` |
| `BUILDKITE_LOCAL_HOOKS_ENABLED` |
| `BUILDKITE_PLUGINS_ENABLED` |
| `BUILDKITE_PLUGINS_PATH` |
| `BUILDKITE_SHELL` |
| `BUILDKITE_SSH_KEYSCAN` |

### Error response for protected variables

```
HTTP/1.1 422 Unprocessable Entity
```

```json
{
  "error": "the following environment variables are protected, and cannot be modified: [BUILDKITE_SHELL]"
}
```

---

## Common error responses

All error responses use the same JSON shape:

<ResponseField name="error" type="string" required>
  A human-readable description of what went wrong.
</ResponseField>

| Status | Cause |
|---|---|
| `400 Bad Request` | Request body is not valid JSON. |
| `401 Unauthorized` | Missing, malformed, or invalid bearer token. |
| `422 Unprocessable Entity` | One or more keys in the request are protected variables, or a `PATCH` request set a value to `null`. |
